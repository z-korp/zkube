# zKube Audio Generation Prompts

> **Music**: Suno AI (3 loopable songs per theme = 30 tracks)
> **Sound Effects**: Audiogen (7 SFX per theme = 70 effects)

## File Structure

```
public/assets/{themeId}/sounds/
  musics/
    track-1.mp3    # Main Theme (menu / home)
    track-2.mp3    # Regular Level (gameplay)
    track-3.mp3    # Boss Level
  effects/
    break.mp3      # Single line clear
    explode.mp3    # Multi-line combo
    move.mp3       # Block slide
    new.mp3        # New row spawns
    start.mp3      # Level start
    swipe.mp3      # Block swipe gesture
    over.mp3       # Game over
```

---

## Global Context for All Prompts

zKube is an on-chain puzzle roguelike. Players slide blocks on an 8x10 grid to clear lines, progress through 50 levels, earn cubes, and fight themed bosses. Think Tetris meets roguelike dungeon crawling. The game is mobile-first, played in portrait. Music must loop seamlessly and not fatigue over long sessions.

# GENERATION TIPS

## Suno Tips
- Add **"instrumental, no vocals"** to every prompt to prevent singing
- Add **"seamless loop"** — Suno often respects this for clean loop points
- Specify **BPM** to control energy level
- Use **"clean mix, mobile-friendly"** to get mixes that sound good on phone speakers
- Generate multiple versions and pick the best loop point
- Trim silence from start/end and crossfade loop points in post-production

## Audiogen Tips
- Keep SFX descriptions focused on **one clear sound event**
- Specify **duration** (Audiogen respects time constraints better with explicit durations)
- Use **"punchy", "clean attack", "fast decay"** for game SFX
- For move/swipe sounds, emphasize **"very short"** and **"non-intrusive"**
- For explode sounds, layer multiple generated variations for richness
- Normalize all SFX to consistent loudness in post-production (-14 LUFS recommended)

## Post-Production Checklist
- [ ] All music tracks loop seamlessly (crossfade edit points if needed)
- [ ] All SFX trimmed tight (no leading/trailing silence)
- [ ] Loudness normalized: music at -14 LUFS, SFX at -12 LUFS
- [ ] Format: MP3 192kbps (good quality/size balance for mobile)
- [ ] File names match catalog exactly (track-1.mp3, break.mp3, etc.)
- [ ] Test on phone speakers — not just headphones
